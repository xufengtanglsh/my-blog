langChain基础-embedding

①安装sentence-transformers
pip install sentence-transformers
误区：即使使用langChain接口调用embedding模型，也必须安装sentence-transformers
因为langChain接口只是对sentence-transformers进行了封装，底层还是需要

②调用BAAI/bge-m3模型，生成embedding

③安装chromadb
python3 -m pip install chromadb

④存入chromadb

get_or_create_collection函数的参数说明：

name：
集合（collection）的名字，字符串，你自己定，相当于 「这张向量表 / 这个库」的标识。
同一 client 里 名字唯一：没有就 创建，已有就 取出来用（get_or_create）。
后面 collection.add / query 都是往这个集合里读写。

metadata:
这是 建集合时带的元数据，用来影响 向量索引怎么算「远近」。
Chroma 默认用 HNSW 做近似最近邻检索。
hnsw:space 表示 距离/相似度空间：
“cosine”：按 余弦距离（与 余弦相似度 对应；向量已 单位化 时，和用内积排序常一致）。
还有常见取值如 “l2”（欧氏距离）、”ip”（内积 / inner product）。
为什么写 cosine：
你对 embedding 做了 normalize_embeddings=True 时，向量在单位球面上，用 cosine 很常见，语义是「方向有多接近」。
注意：
这类 metadata 一般要在集合「第一次创建」时定好；已有集合再改可能受限或需建新集合，以 Chroma 版本文档为准。
若没传，会用 默认距离（不同版本默认可能是 L2，以文档为准）。
